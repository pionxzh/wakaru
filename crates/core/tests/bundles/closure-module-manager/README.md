# Closure ModuleManager fixtures

`synthetic.js` is the small neutral fixture used for general raw/full unpack,
wrapper, and provenance coverage.

`annotated-served-order-shape.js` is a minimized, fully anonymized reproduction
of the structural behavior reported in
[wakaru#195](https://github.com/pionxzh/wakaru/issues/195). It uses synthetic
module IDs, namespace names, loader helpers, and payloads; no production bundle
identity or application payload is retained.

`../closure-module-manager-gen/` complements this with payloads generated by a
pinned Closure Compiler invocation. Its README documents the boundary between
Closure Compiler output and the separate serving pass.

The minimized fixture preserves only the load-bearing shapes:

- a top-level shared-namespace bootstrap and matching `.call(...)` wrapper;
- wrapper-local `var window=this` prelude;
- `_ModuleManager_initialize(graph, loadingIds)` with graph order differing
  from served order;
- paired begin/end loader calls around non-empty modules;
- stacked annotations for synthetic empty modules;
- `_DumpException(e)` guards.
